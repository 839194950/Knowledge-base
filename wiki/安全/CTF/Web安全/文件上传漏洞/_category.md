# 文件上传漏洞

## 概述

文件上传漏洞是 Web 应用程序中常见的安全漏洞之一。当应用程序允许用户上传文件，但没有对上传的文件进行充分的验证和过滤时，攻击者可以上传恶意文件（如 WebShell），进而获取服务器的控制权。

## 漏洞原理

文件上传漏洞的核心问题在于：
- 缺乏对文件类型的严格验证
- 缺乏对文件内容的安全检查
- 上传目录权限配置不当
- 服务器解析漏洞利用

## 常见绕过技术

1. **文件后缀绕过** - 使用黑名单外的后缀（如 .phtml, .php5, .php3）
2. **文件头绕过** - 添加合法文件头（如 GIF89a）
3. **Content-Type 绕过** - 修改 MIME 类型
4. **00 截断** - 利用文件名截断漏洞
5. **双写绕过** - 利用文件名过滤逻辑缺陷
6. **图片马** - 将 WebShell 与图片结合
