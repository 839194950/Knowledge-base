# 文件包含漏洞

## 概述

**文件包含漏洞** 是 Web 应用中常见的安全漏洞之一。当应用程序在包含文件时，如果用户可以控制被包含的文件名，攻击者就可能利用这个漏洞读取任意文件或执行恶意代码。

### 为什么重要
- 可能导致敏感信息泄露
- 可能导致远程代码执行（RCE）
- 在 CTF 比赛中经常出现

---

## 详细解释

### 漏洞原理

文件包含漏洞通常出现在使用动态文件名包含文件的场景中。例如：

```php
<?php
$file = $_GET['file'];
include($file);
?>
```

在这个例子中，用户可以通过 `file` 参数控制被包含的文件名，从而：
- 读取系统中的任意文件（如 `/etc/passwd`）
- 包含远程恶意文件（如果 `allow_url_include` 开启）
- 配合文件上传漏洞 getshell

### 文件包含的类型

1. **本地文件包含（LFI）**：包含服务器本地的文件
2. **远程文件包含（RFI）**：包含远程服务器上的文件（需要 `allow_url_include` 开启）

### 常见的包含函数

在 PHP 中，常见的文件包含函数有：
- `include()`
- `include_once()`
- `require()`
- `require_once()`

这些函数的区别主要在于错误处理和是否重复包含，但在漏洞利用方面基本相同。

---

## 主要利用方式

### 1. 读取敏感文件

```
?file=../../../../etc/passwd
```

### 2. 利用 PHP 伪协议

PHP 提供了多种伪协议，可以用来绕过限制：
- `php://filter` - 读取文件内容（编码后）
- `php://input` - 读取 POST 数据
- `data://` - 执行数据中的代码
- `zip://` / `phar://` - 处理压缩文件

### 3. 配合文件上传

如果目标同时存在文件上传漏洞，可以：
1. 上传包含恶意代码的文件
2. 通过文件包含漏洞执行上传的文件
