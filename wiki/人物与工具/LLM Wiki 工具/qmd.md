# qmd

## 概述

qmd 是一个强大的本地 Markdown 文件搜索引擎，专门为大规模 [[核心概念/LLM Wiki 基础/LLM Wiki]] 设计！它让你的 Wiki 搜索变得更快、更智能！

**简单来说：qmd = 给你的 Wiki 装上超级搜索大脑！**

## 什么是 qmd？

qmd 是一个专门为 Markdown 文件设计的本地搜索引擎，特别适合大规模的 Wiki！

### 核心特点

| 特点 | 说明 |
|------|------|
| **混合搜索** | BM25 + 向量搜索结合 |
| **LLM重排序** | 用 AI 重新排序结果 |
| **完全本地** | 一切都在设备上运行 |
| **CLI接口** | 命令行工具，方便调用 |
| **MCP Server** | 作为 AI 原生工具使用 |

### 形象比喻

想象一下：
- 你的 Wiki = 一个大图书馆
- index.md = 简单的分类目录
- **qmd = 一个超级图书管理员，能瞬间找到你需要的内容！**

## 核心特性详解

### 1. 混合 BM25/向量搜索

什么是混合搜索？

- **BM25** = 传统关键词搜索（找匹配的词）
- **向量搜索** = 语义搜索（找意思相近的）
- **混合** = 两者结合，效果更好！

### 2. LLM 重新排序

这是 qmd 的特别之处！
- 先找到相关结果
- 再用 LLM 重新排序
- 把最相关的放在前面
- **更智能的搜索！**

### 3. 完全在设备上运行

这意味着：
- 隐私性好，数据不会离开你的设备
- 不需要联网
- 响应速度快
- 安全可控

### 4. CLI 接口

CLI 就是命令行接口！
- 让 LLM 可以通过命令行调用
- 可以写脚本自动化
- 灵活易用

### 5. MCP Server

MCP 就是 Model Context Protocol！
- 让 LLM 可以作为原生工具调用
- 无缝集成
- AI 直接就能用

## 适用场景

什么时候需要 qmd？

### 小 Wiki（不需要）

- 只有几十篇笔记
- index.md 就够了
- 简单搜索也能找到

### 大规模 Wiki（推荐使用）

- 几百上千篇笔记
- index.md 不够用了
- 需要强大的搜索功能
- **qmd 就派上用场了！**

### 具体场景

| 场景 | 为什么 qmd 适合 |
|------|----------------|
| **研究项目** | 大量资料需要检索 |
| **学习笔记** | 课程内容多，需要快速找 |
| **个人知识库** | 知识积累多了，需要好搜索 |
| **团队 Wiki** | 多人使用，搜索要求高 |

## 工作原理

让我们看看 qmd 是怎么工作的！

### 搜索流程

1. **你输入查询** → 你要找什么？
2. **BM25搜索** → 找关键词匹配的
3. **向量搜索** → 找语义相近的
4. **LLM重排序** → AI帮你排优先级
5. **返回结果** → 给你最相关的！

## 与 LLM Wiki 的关系

在 [[核心概念/LLM Wiki 基础/LLM Wiki]] 模式中：

- **The Wiki** = 你的知识库
- **qmd** = 搜索这个库的工具
- **完美配合！**

### 示例：LLM 调用 qmd

LLM 可以这样用 qmd：
1. 用户问了一个问题
2. LLM 调用 qmd 搜索相关 Wiki 页面
3. qmd 返回最相关的结果
4. LLM 基于这些结果回答问题
5. 既高效又准确！

## 常见问题

### Q1：qmd 替代了 index.md 吗？

没有！它们是互补的：
- index.md = 浏览和导航
- qmd = 强大的搜索功能
- **一起用，效果最好！**

### Q2：我需要懂技术才能用 qmd 吗？

不需要！qmd 提供了简单的接口，LLM 可以直接调用，你只需要享受搜索结果就行！

### Q3：qmd 只能用于 LLM Wiki 吗？

不是！qmd 可以用于任何 Markdown 文件集合！
