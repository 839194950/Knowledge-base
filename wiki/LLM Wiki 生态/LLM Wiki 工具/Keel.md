# Keel

## 概述

Keel 是一款基于 [[LLM Wiki 生态/LLM Wiki 基础/LLM Wiki]] 模式的 Mac 应用，由 simonsysun 构建！它让你可以用本地优先的方式管理个人知识库！

**简单来说：Keel = 开箱即用的 LLM Wiki 应用！**

## 什么是 Keel？

Keel 是一个完整的、本地优先的 LLM Wiki 应用，专门为 Mac 用户设计！

### 核心特点

| 特点 | 说明 |
|------|------|
| **本地优先** | 数据都在你自己的电脑上 |
| **纯 Markdown** | 知识以纯 Markdown 存储在磁盘上 |
| **模型可交换** | 可以换用不同的 AI 模型 |
| **免费开源** | 完全免费，源代码开放 |
| **自己的上下文** | 你拥有自己的所有数据和上下文 |

### 经典比喻

想象一下：
- 传统笔记应用 = 把数据存在别人的服务器上
- **Keel = 把图书馆建在你自己的书房里！**

## 核心特性详解

### 1. 本地优先（Local-first）

什么是本地优先？
- 数据完全在你的电脑上
- 不需要联网也能用
- 数据完全由你控制
- 不用担心隐私问题

### 2. 纯 Markdown 存储

为什么用纯 Markdown？
- 开放格式，不会被锁定
- 可以用任何 Markdown 编辑器打开
- 用 Git 管理版本
- 将来想换工具也容易

### 3. 模型可交换

你可以：
- 用不同的 AI 模型
- 根据需要选择
- 不会被绑定在一家公司

### 4. 拥有自己的上下文

这是什么意思？
- 你的数据就是你的
- 你的历史对话都是你的
- 你的知识积累都是你的
- 完全可控！

## 适用场景

Keel 适合用在这些地方：

| 场景 | 为什么适合 |
|------|-----------|
| **个人知识库** | 本地管理，安全放心 |
| **学习笔记** | 积累知识，复利增长 |
| **研究工作** | 整理资料，发现关联 |
| **隐私敏感内容** | 完全本地，不会泄露 |

## 与 LLM Wiki 的关系

Keel 是 [[LLM Wiki 生态/LLM Wiki 基础/LLM Wiki]] 模式的一个具体实现！

在 LLM Wiki 三层架构中：
- **Raw sources** = Keel 管理的原始资料
- **The wiki** = Keel 生成的 Markdown
- **The schema** = Keel 的工作逻辑

## 常见问题

### Q1：Keel 只能在 Mac 上用吗？

是的，Keel 是专门为 Mac 设计的应用。

### Q2：我需要自己部署 AI 模型吗？

不需要！Keel 支持模型交换，你可以选择适合自己的方式！

### Q3：数据安全吗？

非常安全！所有数据都在你本地，完全由你控制！
