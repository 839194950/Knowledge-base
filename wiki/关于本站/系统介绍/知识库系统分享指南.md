# 知识库系统分享指南

本文档介绍如何将你的个人知识库系统分享给他人使用。

## 🎯 分享方式概览

| 方式 | 适用场景 | 优点 | 缺点 |
|------|----------|------|------|
| **Vercel 部署分享链接** | 分享给大众访问 | 访问方便、无需安装 | 内容公开（可选私有） |
| **GitHub 仓库分享** | 分享给开发者 | 可协作、可复刻 | 需要技术背景 |
| **压缩包分享** | 离线使用、小范围分享 | 完全控制、离线可用 | 需要自行部署 |
| **仅分享 wiki 内容** | 只分享知识，不分享系统 | 简单、文件小 | 无 Web 界面 |

---

## 🚀 方式一：Vercel 部署分享（推荐）

这是最简单、最常用的分享方式，他人可以通过网页直接访问你的知识库。

### 前置条件

1. 项目已经推送到 GitHub 仓库
2. 拥有 Vercel 账号

### 部署步骤

#### 1. 准备 GitHub 仓库

确保你的代码已经推送到 GitHub：

```bash
# 查看当前状态
git status

# 添加所有更改
git add .

# 提交更改
git commit -m "完善知识库内容"

# 推送到 GitHub
git push
```

#### 2. 在 Vercel 中部署

1. 访问 [Vercel](https://vercel.com) 并使用 GitHub 账号登录
2. 点击 "Add New Project"
3. 选择你的 GitHub 仓库
4. 配置项目（Vercel 会自动检测 Next.js 项目）：
   - **Project Name**: 输入项目名称（如：`my-knowledge-base`）
   - **Framework Preset**: Next.js（自动检测）
   - **Root Directory**: 保持默认
   - **Build Command**: `npm run build`（已配置）
   - **Output Directory**: `.next`（已配置）
5. 点击 "Deploy"
6. 等待部署完成（通常需要 1-2 分钟）

#### 3. 分享链接

部署完成后，Vercel 会提供一个类似 `https://你的项目名.vercel.app` 的链接，你可以：

- 直接分享这个链接给他人
- 在 Vercel 项目设置中配置自定义域名
- 设置为私有项目（付费方案）

### 可选配置

#### 配置自定义域名

1. 在 Vercel 项目设置中找到 "Domains"
2. 添加你的域名（如：`kb.yourdomain.com`）
3. 按照提示配置 DNS 解析

#### 配置环境变量

如果需要，可以在 Vercel 项目设置的 "Environment Variables" 中添加环境变量。

## 💻 方式二：GitHub 仓库分享

适合分享给其他开发者，他们可以复刻、修改和部署自己的版本。

### 仓库准备

1. 确保项目已推送到 GitHub
2. 在 GitHub 仓库中添加详细的 README（已有）
3. 添加开源许可证（可选）

### 分享方式

1. 分享 GitHub 仓库链接
2. 他人可以：
   - **Fork** 仓库到自己账号
   - **Clone** 到本地：
     ```bash
     git clone https://github.com/你的用户名/仓库名.git
     cd 仓库名
     npm install
     npm run dev
     ```
   - 提交 Pull Request 贡献代码
   - 点击 "Use this template" 按钮创建自己的项目（如果设为模板仓库）

### 设为模板仓库

在 GitHub 仓库设置中：

1. 进入 "Settings"
2. 勾选 "Template repository"
3. 这样他人可以直接基于你的仓库创建新项目

## 📦 方式三：压缩包分享

适合离线使用或小范围分享。

### 打包步骤

1. 清理构建文件：
   ```bash
   rm -rf node_modules
   rm -rf .next
   rm -rf out
   ```

2. 打包整个项目：
   - Windows: 右键文件夹 → 发送到 → 压缩(zipped)文件夹
   - macOS/Linux: `zip -r knowledge-base.zip .`

3. 分享压缩包

### 接收者使用步骤

1. 解压压缩包
2. 安装依赖：
   ```bash
   npm install
   ```
3. 本地运行：
   ```bash
   npm run dev
   ```
4. 或构建后运行：
   ```bash
   npm run build
   npm start
   ```

## 📝 方式四：仅分享 wiki 内容

如果只想分享知识库内容，不想分享整个系统。

### 分享方式

1. 直接分享 `wiki/` 文件夹
2. 他人可以：
   - 使用 Obsidian 打开（将 `wiki/` 设为 vault）
   - 使用任何 Markdown 编辑器查看
   - 导入到其他笔记系统

### 打包 wiki 内容

```bash
# 只打包 wiki 文件夹
zip -r wiki-content.zip wiki/
```

## 🔒 隐私考虑

### 公开 vs 私有

- **公开分享**：适合开源知识、教程、公共文档
- **私有分享**：适合个人笔记、敏感内容

### Vercel 隐私设置

- 免费版：项目默认公开
- 付费版：可以设为私有项目

### GitHub 隐私设置

- 创建仓库时选择 Public 或 Private
- Private 仓库需要邀请协作者

## 📚 分享最佳实践

### 1. 清理敏感信息

分享前确保：

- 移除 `.env` 文件中的敏感信息
- 移除 `.trae/` 等本地配置目录
- 检查是否有个人隐私内容

### 2. 完善文档

- 确保 README 清晰易懂
- 添加部署指南
- 添加使用教程（已有）

### 3. 版本控制

- 使用 Git 标签管理版本
- 记录重要变更
- 维护 CHANGELOG

### 4. 社区建设（如果开源）

- 开启 Issues 功能
- 添加贡献指南
- 建立讨论区

## 🎓 接收者入门指南

如果你是接收了分享的用户：

### 快速开始

1. **如果获得 Vercel 链接**：直接在浏览器访问即可
2. **如果获得 GitHub 仓库**：
   ```bash
   git clone https://github.com/用户名/仓库名.git
   cd 仓库名
   npm install
   npm run dev
   ```
3. **如果获得压缩包**：
   - 解压
   - 运行 `npm install`
   - 运行 `npm run dev`
4. **如果只获得 wiki 内容**：
   - 用 Obsidian 或其他 Markdown 编辑器打开

### 创建自己的知识库

1. Fork 或复制项目
2. 清空或替换 `wiki/` 和 `raw/` 中的内容
3. 开始添加自己的内容！
