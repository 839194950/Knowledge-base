# 汇编语言内存管理

## 概述

内存管理是系统编程的重要组成部分。在汇编语言中，你可以直接通过系统调用来动态分配和释放内存，精确控制内存地址，实现高效的内存管理操作。

内存管理的核心能力包括：
- 理解程序在内存中的布局
- 使用系统调用分配和释放内存
- 安全地读写内存
- 管理栈空间

---

## 程序的内存布局

一个运行中的 Linux 程序在内存中有着固定的布局结构。理解这个布局是掌握汇编语言内存管理的基础。

### 内存布局图

### 各区域详解

#### 1. 代码段 (.text)
- **用途：** 存放程序的机器指令
- **特点：** 只读区域，防止程序修改自身代码
- **位置：** 内存低地址区域

#### 2. 数据段 (.data)
- **用途：** 存放已初始化的全局变量和静态变量
- **特点：** 可读可写，程序启动时初始化

#### 3. BSS 段 (.bss)
- **用途：** 存放未初始化的全局变量和静态变量
- **特点：** 程序加载时自动初始化为 0，不占用磁盘空间

#### 4. 堆 (Heap)
- **用途：** 动态内存分配区域
- **特点：** 向上增长，由程序员手动管理
- **管理方式：** brk/sbrk 或 mmap 系统调用

#### 5. 内存映射区域
- **用途：** 内存映射文件、共享库等
- **特点：** 灵活的内存分配方式

#### 6. 栈 (Stack)
- **用途：** 函数调用、局部变量、返回地址
- **特点：** 向下增长，由系统自动管理
- **栈顶指针：** ESP 寄存器指向栈顶

#### 7. 内核空间
- **用途：** 操作系统内核
- **特点：** 用户程序不可直接访问

##### 动态内存分配：brk 系统调用

`sys_brk`（系统调用号 45）是最基础的动态内存分配方式。它通过调整程序的**数据段边界（program break）**来分配和释放内存。

```mermaid
graph TD
    A[brk系统调用] --> B{ebx值}
    B -->|ebx=0| C[获取当前break地址]
    B -->|ebx=新地址| D[设置新break地址]
    C --> E[返回EAX=新break地址]
    D --> E
    F[分配内存] --> G[获取当前break]
    G --> H[增加需要的大小]
    H --> I[设置新break]
    I --> J[返回新内存可用]
    K[释放内存] --> L[恢复到原始break]
```

### 什么是 program break？

program break 是数据段（包括 .data、.bss 和堆）的结束位置。在 program break 之上的内存，程序无法访问。通过移动这个边界，我们就可以扩展或缩小堆的大小。

### brk 系统调用使用方法

| 调用方式 | 说明 | 返回值 |
|---------|------|--------|
| ebx = 0 | 获取当前 break 地址 | EAX = 当前 break 地址 |
| ebx = 新地址 | 设置新的 break 地址 | EAX = 新的 break 地址（失败返回原地址） |

### brk 使用示例

```nasm
; 文件路径：brk_demo.asm
; 使用 sys_brk 动态分配内存

section .data
    alloc_msg db 'Memory allocated successfully at: 0x'
    alloc_len equ $ - alloc_msg
    newline db 0xA

section .bss
    orig_break resd 1  ; 保存原始 break 地址

section .text
global _start

_start:
    ; 1. 获取当前 program break
    mov eax, 45          ; sys_brk
    mov ebx, 0           ; ebx=0 表示查询当前 break
    int 0x80
    mov [orig_break], eax  ; 保存原始 break 地址

    ; 2. 分配 4096 字节（4KB）内存
    mov ebx, eax         ; 当前 break 地址
    add ebx, 4096        ; 增加 4096 字节
    mov eax, 45          ; sys_brk
    int 0x80
    ; EAX = 新的 break 地址（即分配后的区域末尾）

    ; 3. 新分配的内存在 orig_break 到 eax-1 之间
    ; 可以安全地读写这片内存
    mov ebx, [orig_break]  ; 获取分配区域的起始地址
    mov dword [ebx], 42    ; 在新内存中写入 42
    mov eax, [ebx]         ; 读取回来

    ; 4. 释放内存：将 break 恢复到原位
    mov eax, 45          ; sys_brk
    mov ebx, [orig_break]  ; 恢复到原始 break
    int 0x80

    mov eax, 1
    mov ebx, 0
    int 0x80
```

### brk 的局限性

> **重要提示：** sys_brk 只能调整连续的数据段边界，不能释放中间的内存。要释放某块已分配的内存，必须先释放之后分配的所有内存。这就是为什么现代程序更多使用 mmap 或 malloc 库函数。

##### 使用 mmap 分配内存（推荐）

`sys_mmap2`（系统调用号 192）是更灵活的内存分配方式。它可以独立分配和释放多块内存，不受 brk 的连续内存限制。

```mermaid
mindmap
  root((mmap系统调用))
    参数
      ebx addr
      ecx length
      edx prot
      esi flags
      edi fd
      ebp offset
    权限
      PROT_READ
      PROT_WRITE
      PROT_EXEC
      PROT_NONE
    标志
      MAP_PRIVATE
      MAP_ANONYMOUS
    优势
      多块独立内存
      可独立释放
      灵活权限设置
      可映射文件
```

### mmap 的优势

1. 可以分配多块不连续的内存
2. 每块内存可以独立释放
3. 可以设置内存保护权限
4. 可以映射文件到内存

### mmap 系统调用参数

| 寄存器 | 参数 | 说明 |
|--------|------|------|
| ebx | addr | 建议的内存地址（0 表示让内核选择） |
| ecx | length | 分配的内存大小（字节） |
| edx | prot | 内存保护权限 |
| esi | flags | 映射标志 |
| edi | fd | 文件描述符（匿名映射用 -1） |
| ebp | offset | 文件偏移（匿名映射用 0） |

### 常用的 prot 权限值

| 常量 | 值 | 说明 |
|------|----|------|
| PROT_READ | 1 | 可读 |
| PROT_WRITE | 2 | 可写 |
| PROT_EXEC | 4 | 可执行 |
| PROT_NONE | 0 | 不可访问 |

可以使用按位或（|）组合多个权限，例如：`PROT_READ | PROT_WRITE`

### 常用的 flags 标志值

| 常量 | 值 | 说明 |
|------|----|------|
| MAP_PRIVATE | 2 | 私有映射（修改不影响文件） |
| MAP_ANONYMOUS | 0x20 | 匿名映射（不关联文件） |

### mmap 使用示例

```nasm
; 文件路径：mmap_demo.asm
; 使用 mmap 分配可独立释放的内存

section .data
    ; mmap 相关常量
    PROT_READ equ 1
    PROT_WRITE equ 2
    MAP_PRIVATE equ 2
    MAP_ANONYMOUS equ 0x20

section .bss
    mem_block resd 1  ; 保存分配的内存地址

section .text
global _start

_start:
    ; mmap 调用：分配 4096 字节的匿名内存
    mov eax, 192                 ; sys_mmap2
    mov ebx, 0                   ; 让内核选择地址
    mov ecx, 4096                ; 分配大小：4KB
    mov edx, PROT_READ | PROT_WRITE  ; 可读可写
    mov esi, MAP_PRIVATE | MAP_ANONYMOUS  ; 私有匿名映射
    mov edi, -1                  ; 文件描述符（匿名映射用 -1）
    mov ebp, 0                   ; 偏移量（匿名映射用 0）
    int 0x80
    ; EAX = 分配的内存地址（失败返回负值）

    cmp eax, -4096               ; 检查是否失败
    ja mmap_failed               ; 如果在 -4095 ~ -1 之间，失败

    mov [mem_block], eax         ; 保存内存地址

    ; 使用分配的内存：写入数据
    mov ebx, [mem_block]
    mov dword [ebx], 0x12345678     ; 写入 4 字节
    mov dword [ebx + 4], 'runo'     ; 写入 "runo"
    mov dword [ebx + 8], 'ob!!'     ; 写入 "ob!!"

    ; 释放内存：munmap
    mov eax, 91                   ; sys_munmap
    mov ebx, [mem_block]          ; 内存地址
    mov ecx, 4096                 ; 释放大小
    int 0x80

    jmp exit

mmap_failed:
    ; 处理错误...

exit:
    mov eax, 1
    mov ebx, 0
    int 0x80
```

### munmap 释放内存

使用 `sys_munmap`（系统调用号 91）释放 mmap 分配的内存：

```nasm
mov eax, 91     ; sys_munmap
mov ebx, addr   ; 要释放的内存地址
mov ecx, length ; 要释放的大小
int 0x80
```

## 内存读写操作

在汇编语言中，所有内存访问都通过 `mov` 指令配合方括号 `[]` 来完成。

### 基本内存读写

```nasm
; 内存读写基本操作

section .data
    var1 db 0x55      ; 1 字节
    var2 dw 0x1234    ; 2 字节
    var3 dd 0x12345678 ; 4 字节

section .text
global _start

_start:
    ; 读取不同大小的内存
    mov al, [var1]    ; 读取 1 字节：al = 0x55
    mov ax, [var2]    ; 读取 2 字节：ax = 0x1234
    mov eax, [var3]   ; 读取 4 字节：eax = 0x12345678

    ; 写入不同大小的内存
    mov byte [var1], 0xAA   ; 写入 1 字节
    mov word [var2], 0xABCD ; 写入 2 字节
    mov dword [var3], 0xDEADBEEF ; 写入 4 字节

    ; 通过指针操作内存
    mov ebx, var3     ; ebx 指向 var3
    mov eax, [ebx]    ; 间接读取 var3 的值
    add dword [ebx], 1 ; var3 = var3 + 1

    mov eax, 1
    mov ebx, 0
    int 0x80
```

### 数据大小说明

| 关键字 | 大小 | 寄存器示例 |
|--------|------|-----------|
| byte | 1 字节 | al, bl |
| word | 2 字节 | ax, bx |
| dword | 4 字节 | eax, ebx |

## 内存操作的安全规范

在汇编语言中操作内存没有"安全网"，任何错误都可能导致程序崩溃或数据损坏。以下是必须牢记的安全规范：

| 规范 | 说明 | 违反后果 |
|------|------|---------|
| 不越界访问 | 读写不超过变量/缓冲区的大小 | 数据损坏、段错误 |
| 不读未初始化内存 | .bss 段的值不确定 | 不可预测的结果 |
| 不在无效地址上读写 | 确保指针指向有效内存 | 段错误（Segmentation Fault） |
| 不写只读内存 | .text 段和常量区不可写 | 段错误 |
| 地址对齐 | 字访问用偶地址，双字用 4 的倍数 | 性能下降或总线错误 |

### 常见内存错误示例

```nasm
; 常见内存错误示例（警告：会导致崩溃！）

section .data
    small_buf db 0, 0, 0, 0  ; 只有 4 字节

section .text
global _start

_start:
    ; 危险操作1：越界写入
    ; mov dword [small_buf + 3], 0x12345678
    ; 这会覆盖 small_buf 之后 3 字节的数据！

    ; 危险操作2：写入代码段（只读）
    ; mov dword [_start], 0x90  ; 尝试修改代码，会导致段错误

    ; 危险操作3：空指针/无效地址
    ; mov eax, [0]       ; 读取地址 0，会导致段错误
    ; mov [0], eax       ; 写入地址 0，会导致段错误

    ; 安全做法：始终在分配的内存范围内操作
    mov dword [small_buf], 'runo'  ; 正确：在 4 字节内操作

    mov eax, 1
    mov ebx, 0
    int 0x80
```

## 栈内存管理

栈是另一种重要的内存区域，由 PUSH/POP 指令和 ESP 寄存器自动管理。

### 栈的基本操作

```mermaid
sequenceDiagram
    participant ESP as ESP栈指针
    participant Stack as 栈内存
    
    Note over ESP,Stack: PUSH操作
    ESP->>ESP: ESP -= 4
    ESP->>Stack: 写入数据到[ESP]
    
    Note over ESP,Stack: POP操作
    Stack->>ESP: 从[ESP]读取数据
    ESP->>ESP: ESP += 4
    
    Note over ESP,Stack: 分配局部空间
    ESP->>ESP: ESP -= size
    Note over ESP,Stack: 使用栈空间
    ESP->>ESP: ESP += size
```

```nasm
; 栈操作全面示例

section .data
    original_esp dd 0

section .text
global _start

_start:
    mov [original_esp], esp  ; 保存原始栈指针

    ; PUSH：压栈（ESP 减 4，写入数据）
    push dword 100  ; 压入 100
    ; ESP = ESP - 4, [ESP] = 100

    push dword 200  ; 压入 200
    push dword 300  ; 压入 300

    ; 栈布局：
    ; ESP+0: 300
    ; ESP+4: 200
    ; ESP+8: 100

    ; POP：弹栈（读取数据，ESP 加 4）
    pop eax  ; eax = 300, ESP += 4
    pop ebx  ; ebx = 200, ESP += 4
    pop ecx  ; ecx = 100, ESP += 4

    ; 通过栈分配局部空间
    sub esp, 256  ; 在栈上分配 256 字节
    ; 现在 ESP 到 ESP+255 这 256 字节可以安全使用
    mov dword [esp], 42  ; 在局部空间写入 42
    add esp, 256         ; 释放局部空间

    ; 确保 ESP 回到原始位置！
    mov eax, 1
    mov ebx, 0
    int 0x80
```

### 栈操作的重要规则

> **致命警告：** 栈操作最重要的规则是 PUSH 和 POP 必须配对。函数返回前必须确保 ESP 回到正确的位置，否则 RET 会弹出错误的返回地址，导致程序崩溃或执行任意代码。这是汇编编程中最致命的 bug 之一！

栈操作的最佳实践：
1. 保存原始 ESP
2. PUSH 和 POP 数量相等
3. 使用 EBP 建立栈帧（更安全的方式）
4. 函数返回前恢复 ESP

## 内存管理方案对比

不同的内存管理方案有不同的特点，适用于不同的场景：

| 方案 | 灵活性 | 复杂度 | 适用场景 |
|------|--------|--------|---------|
| 全局变量（.data/.bss） | 低（编译时固定） | 最低 | 固定大小的数据 |
| 栈分配（sub esp） | 中（函数内动态） | 低 | 函数局部变量 |
| brk/sbrk | 中（只能增长/收缩） | 中 | 连续内存区域 |
| mmap | 高（任意分配释放） | 高 | 需要灵活管理内存时 |

### 选择建议

- **简单固定数据：** 使用全局变量
- **函数局部变量：** 使用栈分配
- **简单连续内存：** 使用 brk
- **灵活内存管理：** 使用 mmap（推荐）
- **高级应用：** 使用 C 库的 malloc/free
