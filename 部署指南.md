# 个人知识库网站 - 部署指南

## 项目概述

这是一个基于 Next.js 的个人知识库网站，支持 Markdown 渲染、知识图谱、搜索等功能。

## 部署到 GitHub

### 1. 在 GitHub 上创建新仓库

1. 访问 [GitHub](https://github.com) 并登录
2. 点击右上角的 "+" 按钮，选择 "New repository"
3. 填写仓库名称（例如：`personal-knowledge-base`）
4. 选择 Public 或 Private
5. **不要**初始化 README、.gitignore 或 LICENSE（我们已经有了）
6. 点击 "Create repository"

### 2. 推送代码到 GitHub

在本地项目目录中运行以下命令（替换为你的 GitHub 用户名和仓库名）：

```bash
git remote add origin https://github.com/你的用户名/仓库名.git
git branch -M main
git push -u origin main
```

## 部署到 Vercel

### 方法一：通过 Vercel 网站部署（推荐）

1. 访问 [Vercel](https://vercel.com) 并使用 GitHub 账号登录
2. 点击 "Add New Project"
3. 选择你的 GitHub 仓库
4. 配置项目：
   - **Project Name**: 你的项目名称
   - **Framework Preset**: Next.js（会自动检测）
   - **Root Directory**: 保持默认
   - **Build Command**: `npm run build`（已配置）
   - **Output Directory**: `.next`（已配置）
5. 点击 "Deploy"
6. 等待部署完成（通常需要 1-2 分钟）

### 方法二：通过 Vercel CLI 部署

1. 安装 Vercel CLI：
```bash
npm install -g vercel
```

2. 登录 Vercel：
```bash
vercel login
```

3. 部署项目：
```bash
vercel
```

4. 按照提示完成配置

## Vercel 配置

项目已经包含了 `vercel.json` 配置文件：

```json
{
  "buildCommand": "npm run build",
  "outputDirectory": ".next",
  "framework": "nextjs"
}
```

## 本地开发

```bash
# 安装依赖
npm install

# 启动开发服务器
npm run dev

# 构建生产版本
npm run build

# 启动生产服务器
npm start
```

## 项目结构

```
个人知识库网站/
├── wiki/              # 知识库内容（Markdown 文件）
├── raw/               # 原始内容源文件
├── src/               # 源代码
│   ├── app/           # Next.js App Router
│   ├── components/    # React 组件
│   └── lib/           # 工具函数
├── public/            # 静态资源
├── package.json       # 项目配置
└── vercel.json        # Vercel 配置
```

## 注意事项

1. **Git 仓库已初始化**：项目已经有 `.git` 目录，可以直接推送到 GitHub
2. **构建已验证**：项目已通过 `npm run build` 验证，可以正常构建
3. **Vercel 配置已就绪**：`vercel.json` 已配置好
4. **.gitignore 已完善**：不会提交不必要的文件（如 node_modules、.next 等）
